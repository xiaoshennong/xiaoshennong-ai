# 小神农中医AI - 人力成本优化方案

## 核心原则：机器做机器的事，人做人的事

| 机器做 | 人做 |
|--------|------|
| 网上已有数字化资源的抓取 | 网上找不到的稀缺古籍扫描 |
| 文本化古籍的自动结构化 | OCR结果的校验和纠错 |
| 向量化和入库 | 医学内容的审核和标注 |
| 标准化症状匹配 | 复杂辨证的人工复核 |

## 一、古籍数据采集策略：爬虫+人工分工

### 1.1 自动化抓取（机器做）

| 汉典 | zdic.net | 汉字+古籍 | 网页解析 |

**已实现的自动化工具：**

## 二、数据标注流程优化

### 2.1 机器预处理（自动完成）

```
原始数据 → 自动分章 → 自动分段 → 自动生成ID → 自动向量化 → 入库
```

**已实现：**
- `data_pipeline.py` 自动处理JSON/TXT文件
- 智能分章（按章节标题、空行、字数）
- 自动生成结构化ID（书名_篇章_序号）
- 自动调用嵌入模型生成向量

### 2.2 人工只做校验（质量控制）

**学生团队工作清单：**

| 任务 | 工作量 | 说明 |
|------|--------|------|
| OCR结果校验 | 15部古籍 | 只校验机器抓不到的古籍 |
| 章节划分检查 | 100部 | 检查自动分章是否合理 |
| 原文校对 | 30部核心 | 重点校验P0级古籍 |
| 出处标注审核 | 全部 | 确保每段都有准确出处 |

**标注SOP（标准操作流程）：**

```
1. 打开自动生成的JSON文件
2. 对照原始扫描件/可靠版本
3. 检查：
   □ 章节标题是否正确
   □ 原文是否有OCR错误（重点关注生僻字）
   □ 出处标注是否准确（书名+篇章）
4. 如有错误，直接修改JSON文件
5. 标记审核状态：verified / needs_fix
```

**质量控制机制：**

```python
# 自动校验规则
1. 检查是否有空文本段
2. 检查章节标题是否重复
3. 检查文本长度异常（过长/过短）
4. 标记可疑OCR字符（■、□、?等）
5. 生成待人工复核清单
```

## 三、LLM层优化：用Kimi Code API

### 3.1 为什么用Kimi Code API

| 对比 | 本地部署Qwen-7B | DeepSeek API | Kimi Code API |
|------|----------------|-------------|---------------|
| 硬件成本 | 需RTX 4090（2万元） | 无需硬件 | 无需硬件 |
| 月运营成本 | 400元电费 | 300-500元 | 按量计费 |
| 中文能力 | 良好 | 优秀 | 优秀 |
| RAG适配 | 需微调 | 原生支持 | 原生支持 |
| 幻觉控制 | 需额外优化 | 良好 | 良好 |
| 维护成本 | 高（需运维） | 低 | 低 |

**结论：MVP阶段用Kimi Code API最优**
- 无需购买显卡
- 无需维护模型
- 按token计费，初期成本低
- 中文能力强，适合中医古籍

**使用方式：**

```bash

```

└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  第二步：人工处理稀缺古籍（节省92.5%人力）                  │
│  学生团队：                                               │
│  1. 根据清单找扫描件/PDF                                  │
│  2. 用PaddleOCR批量识别                                   │
│  3. 人工校验（重点：生僻字、章节标题）                      │
│  4. 保存为JSON格式                                        │
└─────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────┐
│  第三步：机器自动入库（0人力）                              │
│  python backend/data_pipeline.py --import-all             │

│  第四步：人工质量抽检（10%样本）                            │
│  专家顾问：                                               │
│  1. 随机抽取10%数据                                       │
│  2. 检查溯源准确性                                        │

# 小神农中医AI - MVP启动执行文档

## 一、项目一句话定位

国内首个纯中医古籍权威数据训练、可溯源无幻觉的中医AI辅助平台，MVP阶段聚焦"AI辨证+药食同源养生商城"，验证"可溯源建立信任→养生品快速变现"的核心假设。

---

## 二、MVP阶段（0-3个月）核心目标

| 目标 | 数值 | 验证什么 |
|------|------|---------|
| 完成核心100部古籍数字化 | 100部 | 数据壁垒可构建 |
| 上线AI可溯源辨证功能 | 内测可用 | 技术方案可行 |
| 上线药食同源养生商城 | 5款爆款 | 变现路径可行 |
| 首月订单 | 1000单 | 市场需求真实 |
| 首月营收 | 5万元 | 单位经济模型成立 |
| 毛利率 | ≥25% | 商业模式可持续 |

---

## 三、MVP功能范围（严格砍需求）

### 必须做（P0）
1. **AI可溯源辨证系统**
   - 引导式问诊（20-30个标准化问题）
   - 输出体质类型+调理建议
   - 每条结论标注古籍出处（书名+篇章+原文）
   - 无依据时拒绝回答"暂无权威数据支持"
   
2. **药食同源养生商城**
   - 5款基础爆款：酸梅汤、祛湿茶、清补凉、四神汤、菊花茶
   - 每款产品标注"配方出处+专家审核"
   - 支持微信支付、快递配送
   - AI辨证结果页自动推荐对应养生品

### 不做（等PMF验证后）
- ❌ B端医馆SaaS系统
- ❌ 完整处方流转闭环
- ❌ 区块链存证
- ❌ 疗效反馈闭环
- ❌ 集采供应链
- ❌ 会员体系

## 六、技术架构（MVP最小版本）

### 6.1 整体架构

```
用户（微信小程序）
    ↓
API网关（Nginx + Python Flask/FastAPI）
    ↓
RAG服务层
  ├── 问诊接口：接收症状 → 检索古籍 → 生成辨证结果
  ├── 商城接口：商品列表、订单、支付
  └── 用户接口：登录、健康档案（简化版）
    ↓
数据层
  ├── 向量数据库（Chroma）：古籍文本向量
  ├── 关系数据库（PostgreSQL）：方剂、用户、订单
  └── 文件存储：古籍扫描原图
    ↓
AI层
  ├── 嵌入模型（bge-m3）：本地部署
  ├── 重排序模型（bge-reranker-large）：本地部署
  └── 大模型（DeepSeek V3 API）：生成回答
```

### 6.2 核心RAG流程

```
用户输入症状 → 症状标准化（中医术语词典）
    ↓
向量检索（bge-m3）：召回Top20相关古籍片段
关键词检索（PostgreSQL全文检索）：补充精确匹配
    ↓
重排序（bge-reranker-large）：筛选Top3-5最相关片段
    ↓
构建提示词（强制溯源约束）→ 调用DeepSeek V3 API
    ↓
输出校验（出处匹配检查）→ 返回用户
```

### 6.3 强制溯源提示词模板

```
你是小神农中医AI助手，必须严格遵守以下规则：

1. 只基于下面提供的【参考资料】回答问题，绝对不能使用你自己的任何知识。
2. 如果【参考资料】中没有相关内容，直接回答："抱歉，暂无权威数据支持该问题。"
3. 每给出一个结论，必须在后面标注对应的出处，格式为：[出处：《书名》·篇章]。
4. 不要进行任何推理、演绎或扩展，只复述【参考资料】中的内容。
5. 不要使用任何医疗术语以外的语言，保持专业、简洁。
6. 明确提示：本结果仅供参考，不能替代专业医生诊断。

【参考资料】
{检索到的知识片段，每个片段前标注出处}

用户问题：{用户的提问}
```

### 7.2 包装文案规范（合规）

```
正面：
- 产品名称：小神农·酸梅汤（养生茶饮）
- 显著标注："中国中医科学院 XX 团队配方监制"
- 净含量、生产日期、保质期

背面：
- 配料表：乌梅、山楂、甘草、陈皮、冰糖...
- 配方出处："源自《本草纲目》卷三十六·果部"
- 专家签名/审核标识
- 产地溯源信息
- 食用方法：冲泡或熬煮
- ⚠️ 明确标注："本品为普通预包装食品，不能替代药物治疗"

禁止出现：
- ❌ 治疗、治愈、疗效、有效率等医疗术语
- ❌ 专家医师执业证号
- ❌ 任何疾病名称+产品关联
```

### 7.3 AI推荐转化路径

```
用户完成AI辨证 → 结果显示体质类型
    ↓
结果页底部推荐："根据您的体质，推荐以下养生方"
    ↓
湿热体质 → 推荐祛湿茶（标注"适合湿热体质"）
气虚体质 → 推荐四神汤（标注"适合气虚体质"）
阴虚体质 → 推荐菊花枸杞茶（标注"适合阴虚体质"）
    ↓
点击推荐卡片 → 进入商品详情页 → 一键下单
```

## 十、成功标准与下一步决策

### MVP阶段成功标准（3个月）
- ✅ 100部核心古籍完成数字化和结构化
- ✅ AI辨证功能内测可用，溯源准确率≥80%
- ✅ 养生商城上线，首月1000单，毛利率≥25%
- ✅ 用户NPS≥30（愿意推荐给朋友）

### 达标后进入PMF验证期（3-9个月）
- 上线完整处方流转（对接10-15家医馆）
- 上线B端医馆SaaS（简化版）
- 启动种子轮融资

### 未达标则调整
- 数据质量不够 → 加大专家审核投入
- 转化率低 → 优化AI推荐算法和商品详情页
- 供应链问题 → 切换轻资产分销模式

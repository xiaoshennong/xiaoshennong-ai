# 小神农中医AI · 多 Agent 知识图谱扩展工作流

> **目标**：用多 Agent 并行协作，持续丰富知识图谱，严格遵循"网上搜索 → 批判验证 → 古籍佐证 → 审慎入库"的铁律。

---

## 一、Agent 分工总览

| Agent | 职责 | 输入 | 输出 |
|---|---|---|---|
| **搜索 Agent** | 网上检索临床数据、医案、研究论文 | 症状/药物/方剂名称 | 候选文献列表（含 URL、摘要、来源） |
| **批判 Agent** | 三阶九维可信度评估 | 搜索 Agent 的候选文献 | A/B/C/D 评级报告 |
| **古籍 Agent** | 在中医知识库中寻找佐证 | 待验证结论 | 强佐证/弱佐证/矛盾/无记载 |
| **症状 Agent** | 生成/更新症状节点 SN 编码 | 临床描述、古籍条文 | `symptoms/SN-XX-X-XXX.json` |
| **药物 Agent** | 生成/更新药物节点 DR 编码 | 方剂组成、副作用、临床统计 | `drugs/DR-XX-XXX.json` |
| **方剂 Agent** | 生成/更新方剂节点 FP 编码 | 古籍原文、组成、主治 | `formulas/FP-XX-XXX.json` |
| **副作用 Agent** | 建立药物-副作用关系 | 患者反馈、文献、共现分析 | `adverse/AD-DR-XX-XXX.json` |
| **共现 Agent** | 分析症状共现强度 | 患者档案、方剂主治 | `cooccur/cooccur.json` |
| **聚类 Agent** | 相似患者聚类 | 患者档案 | `clusters/CL-XX-XXX.json` |
| **入库 Agent** | 最终审核、编号分配、文件归档 | 所有 Agent 输出 | 合并到 `data/knowledge_graph/` |

## 二、六步工作流（不可跳步）

```
Step 1: 需求触发
    └─ 用户提问 / 新数据发现 / 图谱缺口识别
           ↓
Step 2: 多源搜索（搜索 Agent 并行）
    ├─ PubMed/CNKI/万方/知网 临床文献
    ├─ 中医古籍数据库
    ├─ 权威医案平台
    └─ 公开不良反应数据库
           ↓
Step 3: 批判性思维验证（批判 Agent）
    ├─ 来源可信度：期刊级别、作者资质、发表时间
    ├─ 内容可信度：证据等级、样本量、一致性
    └─ 适用可信度：症状匹配、人群匹配、冲突检测
           ↓
Step 4: 知识库佐证（古籍 Agent + 已有图谱）
    ├─ 强佐证：古籍原文直接支持
    ├─ 弱佐证：古籍间接相关
    ├─ 矛盾：与古籍冲突 → 标记争议
    └─ 无记载：现代医学/新药数据 → 降低证据等级
           ↓
Step 5: 编号与效果统计
    ├─ 症状 → 分配 SN-部位-类别-序号
    ├─ 药物 → 分配 DR-类别-序号，统计有效率/样本量/副作用
    ├─ 方剂 → 分配 FP-病位-序号
    └─ 副作用 → 分配 AD-DR-药物-序号
           ↓
Step 6: 审慎入库（入库 Agent 人工审批）
    ├─ 初筛：自动去重、格式校验
    ├─ 人工审核：中医专家确认
    ├─ 编号分配：确保唯一性
    └─ 归档：写入 `data/knowledge_graph/`
```

## 三、三阶九维可信度评估

| 维度 | 评分 1-5 | 说明 |
|---|---|---|
| **来源可信度** | | |
| 来源权威性 | 1-5 | 核心期刊/官方指南=5；公众号/论坛=1 |
| 作者资质 | 1-5 | 三甲医院/大学=5；匿名=1 |
| 发表时间 | 1-5 | 近5年=5；20年前=2 |
| **内容可信度** | | |
| 证据等级 | 1-5 | RCT/Meta=5；个案=2 |
| 样本量 | 1-5 | ≥100=5；<20=2 |
| 一致性 | 1-5 | 多研究一致=5；矛盾=1 |
| **适用可信度** | | |
| 症状匹配度 | 1-5 | 完全匹配=5；边缘相关=2 |
| 人群匹配度 | 1-5 | 目标人群一致=5；差异大=1 |
| 冲突检测 | 1-5 | 无冲突=5；与禁忌冲突=1 |

**评级标准**：
- ≥4.5 → **A级**（采用）
- 3.5-4.4 → **B级**（采用，标注局限）
- 2.5-3.4 → **C级**（仅参考）
- <2.5 → **D级**（排除）

## 四、编码体系（强制唯一）

### 症状编码：`SN-部位-类别-序号`

| 部位代码 | 含义 | 类别代码 | 含义 |
|---|---|---|---|
| QT | 全身/一般 | Z | 自觉症状 |
| HD | 头面 | T | 体征 |
| TH | 胸腹 | F | 复合症状 |
| XG | 消化 | | |
| HX | 呼吸 | | |
| SJ | 精神神经 | | |
| MN | 泌尿生殖 | | |

示例：`SN-SJ-Z-001` = 失眠

### 药物编码：`DR-功效类别-序号`

| 类别代码 | 含义 | 类别代码 | 含义 |
|---|---|---|---|
| JB | 解表药 | QR | 清热药 |

### 副作用编码：`AD-DR-药物编码-序号`

示例：`AD-DR-AS-001-001` = 酸枣仁的第1条副作用

| 维度 | 来源1 | 来源2 | 来源3 |

| 症状匹配 | 5 | 5 | 5 |
| **综合评级** | **B** | **A** | **C** |

### 5.3 古籍 Agent 佐证

- **强佐证**：《金匮要略·血痹虚劳病脉证并治》"虚劳虚烦不得眠，酸枣仁汤主之"

## 六、副作用辨别：并发症 vs 药物副作用

当患者出现新症状时，启动以下决策树：

```
新症状出现
    │
    ├─ 时间窗口分析 ──→ <30分钟急性 / 1-3天常见 / >7天慢性
    │
    ├─ 药物副作用库查询 ──→ 已知副作用？发生率？
    │
    ├─ 病情进展路径分析 ──→ 属于当前证型？属于并发症路径？
    │
    ├─ 症状共现网络分析 ──→ 与现有症状共现强度？
    │
    └─ 相似患者验证 ──→ 同方同反应比例？
           ↓
    综合评分（0-120分）
           ↓
    ≥80分：很可能副作用
    50-79分：可能相关，需观察
    <50分：可能病情进展 / coincidental
```

示例：患者服用酸枣仁汤后第3天腹泻
- 时间：1-3天常见副作用窗口 → +25分
- 药物证据：酸枣仁已知腹泻发生率3.2% → +25分
- 排除进展：心阴虚证型不含腹泻 → +20分
- 共现异常：失眠+腹泻低共现 → +15分
- **总分 85 → 很可能副作用**

## 八、行为铁律

1. **零幻觉**：任何结论必须有来源，禁止编造数据。
2. **强制溯源**：每个药物功效、副作用、有效率必须标注来源。
3. **双源验证**：至少一个古籍来源 + 一个临床来源才能评为 A/B 级。
4. **禁忌必提**：十八反、十九畏、妊娠禁忌必须自动阻断并告警。
5. **数据入库审批**：自动生成的 C 级数据必须人工审核后才能提升为 A/B 级。
